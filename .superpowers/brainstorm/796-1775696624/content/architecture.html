Site Architecture

Next.js App Router · PostgreSQL · Prisma · Future-ready for ticket booking

Frontend — Next.js App Router

app/
layout.tsx — shared nav + footer (Warm Stone theme)
page.tsx — landing hero (full-screen DMZ imagery)
history/page.tsx — timeline of the DMZ
explore/page.tsx — interactive map / photo gallery
proposal/page.tsx — amphitheater concept + renders
events/page.tsx — upcoming events + ticket booking
api/
tickets/route.ts — book / list tickets
auth/route.ts — user registration + login

Data Layer

Database

PostgreSQL + Prisma ORM

Type-safe queries, easy migrations, works great with Next.js API routes. Hosted on Neon (free tier) or Supabase.

User { id, name, email, passwordHash }
Event { id, title, date, capacity }
Ticket { id, userId, eventId, seat }
Auth

NextAuth.js

Email + password login to start. Can add Google/GitHub OAuth later. Session stored server-side — secure and simple.

Validation

Zod

All API inputs validated with Zod schemas. Shared types between frontend forms and API routes.

Deployment

Vercel for the Next.js app (free tier, automatic deploys from GitHub) · Neon or Supabase for PostgreSQL (free tier) · .env.local for secrets — never committed
